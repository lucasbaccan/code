Git e .gitignore#

Quando criamos um novo projeto, é uma boa pratica utilizar uma ferramenta de gerenciamento de código, a mais utilizada atualmente é o Git. Quando rodamos git ini, é criado uma pasta .git. Além desta pasta, podemos criar um arquivo .gitignore, do qual informamos quais arquivos e diretórios não queremos que o Git gerencie.

A forma que o Git utiliza para controlar quais arquivos ele não deve controlar é utilizando um arquivo .gitignore. Este arquivo deve ficar no mesmo local em que o arquivo .git está localizado.

Como utilizar#

Quando eu crio um projeto novo, utilizo o site https://gitignore.io/ para gerar o arquivo .gitignore, basta informar qual a linguagem ou ferramenta está utilizando e o site gera um arquivo de exemplo.

O arquivo gerado é o seguinte, nele contem varias configurações para evitar arquivos da linguagem java.

# Created by https://www.toptal.com/developers/gitignore/api/java
# Edit at https://www.toptal.com/developers/gitignore?templates=java
### Java ###
# Compiled class file
*.class
# Log file
*.log
# BlueJ files
*.ctxt
# Mobile Tools for Java (J2ME)
.mtj.tmp/
# Package Files #
*.jar
*.war
*.nar
*.ear
*.zip
*.tar.gz
*.rar

Exemplo de múltiplas linguagens#

O site contem centenas de opções, e você pode juntar mais de uma opção ao mesmo tempo.

Então explore as opções sem medo, aposto que vai achar uma solução para seu projeto.
